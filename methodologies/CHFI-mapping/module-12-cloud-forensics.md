## Module 12: Cloud Forensics

### Understand Cloud Computing Concepts
Cloud forensics is challenged by **multi-tenancy** (data from different customers co-mingled on physical hardware) and lack of direct physical access to hardware storage. Instance memory (RAM) is highly volatile and disappears upon reboot.

### Understand Cloud Forensics
Investigators must deploy specialized tools to capture remote virtual machine snapshots (forensically sound copies of the virtual disk image) and network traffic without interrupting cloud availability.

### Understand Amazon Web Services (AWS) Fundamentals & Perform AWS Forensics
AWS uses **CloudTrail** for comprehensive API logging and monitoring, and **CloudWatch** for centralized log management and metric tracking. Forensics involves generating snapshots of EC2 instances.

### Understand Microsoft Azure Fundamentals & Perform Microsoft Azure Forensics
Utilizing the Azure Portal to generate OS disk snapshots and collect logging metrics for virtual machines and managed services.

### Understand Google Cloud Fundamentals & Perform Google Cloud Forensics
Utilizing the GCP Console to isolate affected instances and secure forensic copies of Google Cloud environments.

**Tools Used in Module 12:** AWS Management Console, Azure Portal, Google Cloud Platform (GCP) Console.
