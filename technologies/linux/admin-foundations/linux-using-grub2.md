# linux-using-grub2

## Using GRUB2 in a Linux System
GRUB2 (GRand Unified Bootloader version 2) is the default bootloader for most modern Linux distributions. It is responsible for loading the Linux kernel and initial RAM disk before handing control over to the operating system. Understanding GRUB2's configuration files and their locations is essential for managing and troubleshooting the boot process.
### Main Configuration Files and Their Locations
1. **/boot/grub2/grub.cfg**
   - **Primary Configuration File**: This is the main configuration file for GRUB2. It contains the boot menu entries and settings. The file is automatically generated by the `grub2-mkconfig` command using templates from `/etc/grub.d` and settings from `/etc/default/grub`.
   - **Location**: `/boot/grub2/grub.cfg` (on BIOS systems) or `/boot/efi/EFI/<distro>/grub.cfg` (on UEFI systems)[1](https://opensource.com/article/17/3/introduction-grub2-configuration-linux).

2. **/etc/default/grub**
   - **User Configuration File**: This file contains user-defined settings that control the behavior of GRUB2. It includes parameters such as the default boot entry, timeout, and kernel parameters.
   - **Location**: `/etc/default/grub`[2](https://access.redhat.com/sites/default/files/attachments/understanding_grub_10_21_2015.pdf).

3. **/etc/grub.d/**
   - **Script Directory**: This directory contains scripts that are executed by `grub2-mkconfig` to generate `grub.cfg`. Each script is responsible for creating specific sections of the configuration file, such as detecting Linux kernels or adding custom menu entries.
   - **Location**: `/etc/grub.d/`[2](https://access.redhat.com/sites/default/files/attachments/understanding_grub_10_21_2015.pdf).

4. **/boot/grub2/grubenv**
   - **Environment Block**: This file stores GRUB2 environment variables, such as the saved default boot entry. It allows GRUB2 to remember settings across reboots.
   - **Location**: `/boot/grub2/grubenv`[1](https://opensource.com/article/17/3/introduction-grub2-configuration-linux).

5. **/boot/efi/EFI/**
   - **UEFI Configuration**: On UEFI systems, GRUB2 stores its configuration and binary files in the EFI system partition. This directory contains the GRUB2 bootloader and configuration files specific to UEFI.
   - **Location**: `/boot/efi/EFI/<distro>/`[2](https://access.redhat.com/sites/default/files/attachments/understanding_grub_10_21_2015.pdf).
### Key Commands for Managing GRUB2
- **Updating GRUB Configuration**: After making changes to `/etc/default/grub` or adding scripts to `/etc/grub.d/`, you need to regenerate `grub.cfg` using:
  ```
  sudo grub2-mkconfig -o /boot/grub2/grub.cfg
  ```

- **Installing GRUB2**: To install GRUB2 to the MBR or UEFI partition, use:
  ```
  sudo grub2-install /dev/sdX
  ```
  Replace `/dev/sdX` with the appropriate device identifier.

Understanding these configuration files and their locations helps in effectively managing the GRUB2 bootloader, ensuring a smooth and customizable boot process for your Linux system.
