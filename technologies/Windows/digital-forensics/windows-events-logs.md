# windows-events-logs
**Windows Event Logs** are a crucial part of the Windows operating system, providing detailed records of system, security, and application events. These logs help administrators diagnose system problems, predict future issues, and investigate security incidents.

Understanding Windows Event Logs is essential for effective system administration and security monitoring. By knowing what each log contains and where to find them, you can better diagnose issues, predict potential problems, and ensure the security of your systems.

## Types of Windows Event Logs
**1. System Logs**
   - **Description**: These logs contain information about the Windows operating system and its components. They record events such as system errors, warnings, and informational messages.
   - **Examples**: Failure to load a boot-start driver, system shutdowns, and hardware failures.
   - **Directory and File Name**: `C:\Windows\System32\winevt\Logs\System.evtx`

**2. Application Logs**
   - **Description**: These logs record events related to software applications installed on the system. They include errors, warnings, and informational messages generated by applications.
   - **Examples**: Crashes of applications like Microsoft Word, errors during software installations.
   - **Directory and File Name**: `C:\Windows\System32\winevt\Logs\Application.evtx`

**3. Security Logs**
   - **Description**: These logs contain records of security-related events, such as login attempts and resource access. They are crucial for auditing and monitoring security.
   - **Examples**: Successful and failed login attempts, changes to user privileges.
   - **Directory and File Name**: `C:\Windows\System32\winevt\Logs\Security.evtx`

**4. Setup Logs**
   - **Description**: These logs record events related to the installation and setup of the operating system and its components.
   - **Examples**: Events related to Active Directory on a domain controller, installation of system updates.
   - **Directory and File Name**: `C:\Windows\System32\winevt\Logs\Setup.evtx`

**5. Forwarded Events**
   - **Description**: These logs collect events that are forwarded from other computers. This is useful in a centralized logging environment.
   - **Examples**: Events from other networked computers for centralized monitoring.
   - **Directory and File Name**: `C:\Windows\System32\winevt\Logs\ForwardedEvents.evtx`

## Accessing and Reading Event Logs
To access and read these logs, you can use the **Windows Event Viewer**:
- **Steps**:
  1. Press `Win + R` to open the Run dialog.
  2. Type `eventvwr` and press Enter.
  3. In the Event Viewer, navigate through the different log categories to view the events.

Each event log entry contains several fields:
- **Log Name/Key**: Type of logging component.
- **Level**: Severity of the event (e.g., critical, error, warning, information).
- **Date/Time**: When the event was recorded.
- **Source**: Component that triggered the event (e.g., application or process name).
- **EventID**: Unique identifier for the event.
- **Task Category**: Additional information about the event.
- **User**: Username of the person logged in when the event was recorded.
- **Computer**: Name of the computer that logged the event.

## Some common Windows Event IDs and their descriptions
1. **Event ID 4624** - An account was successfully logged on.
2. **Event ID 4625** - An account failed to log on.
3. **Event ID 4634** - An account was logged off.
4. **Event ID 4648** - A logon was attempted using explicit credentials.
5. **Event ID 4672** - Special privileges assigned to new logon.
6. **Event ID 4688** - A new process has been created.
7. **Event ID 4697** - A service was installed in the system.
8. **Event ID 1102** - The audit log was cleared.
9. **Event ID 4608** - Windows is starting up.
10. **Event ID 4609** - Windows is shutting down¹²³.

These events are useful for monitoring system activities and security-related events.
